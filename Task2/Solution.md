# 🐳 Basic Docker Commands Cheat Sheet

## 1️⃣ Build Docker Image

```bash
docker build -t myapp .
```

Purpose:
Builds a Docker image from the Dockerfile in the current directory.

---

## 2️⃣ List Docker Images

```bash
docker images
```

Purpose:
Displays all Docker images available on the local system.

---

## 3️⃣ Run Docker Container

```bash
docker run -d -p 3000:3000 myapp
```

Purpose:
Runs a container from the Docker image.

- `-d` → Run in background
- `-p` → Port mapping

---

## 4️⃣ Check Running Containers

```bash
docker ps
```

Purpose:
Shows currently running containers.

---

## 5️⃣ Check All Containers

```bash
docker ps -a
```

Purpose:
Displays all containers including stopped containers.

---

## 6️⃣ View Container Logs

```bash
docker logs <container_id>
```

Example:

```bash
docker logs a1b2c3d4
```

Purpose:
Displays logs/output generated by the container.

---

## 7️⃣ Stop Container

```bash
docker stop <container_id>
```

Purpose:
Stops a running container.

---

## 8️⃣ Start Container Again

```bash
docker start <container_id>
```

Purpose:
Starts a stopped container.

---

## 9️⃣ Remove Container

```bash
docker rm <container_id>
```

Purpose:
Deletes a stopped container.

---

## 🔟 Remove Docker Image

```bash
docker rmi <image_id>
```

Purpose:
Deletes a Docker image from the local system.

---

## 1️⃣1️⃣ Push Image to Docker Hub

```bash
docker push username/myapp
```

Purpose:
Uploads Docker image to Docker Hub repository.

---

## 1️⃣2️⃣ Pull Image from Docker Hub

```bash
docker pull username/myapp
```

Purpose:
Downloads Docker image from Docker Hub.

---

## 1️⃣3️⃣ Docker Init

```bash
docker init
```

Purpose:
Automatically generates Docker configuration files like:
- Dockerfile
- .dockerignore
- compose.yaml

---

## 1️⃣4️⃣ Execute Command Inside Container

```bash
docker exec -it <container_id> bash
```

Purpose:
Opens terminal access inside a running container.

---

## 1️⃣5️⃣ Check Docker Version

```bash
docker --version
```

Purpose:
Displays installed Docker version.

---

## 📚 Key Learnings

- Docker simplifies application deployment
- Containers ensure environment consistency
- Docker images are portable
- Docker Hub acts as a remote repository
- `docker init` makes Dockerization easier for beginners
